## 一句话定位

**为没有司机的 Robotaxi 乘客设计一颗橙色桔子「小桔」，提供安心感、陪伴感和应急保护——它不是一个角色，而是滴滴 DNA 本身的化身。**

差异化核心：
- Robotaxi 乘客是「专注用户」——双手双眼空闲，可承载 5–10 分钟深度交互。这是 NOMI / 小 P 等面向司机的车载 AI 永远做不到的结构性差异化机会。
- 形象 IP 选择「桔子小桔」而非"猫咪/虚拟人/抽象光滴"——因为它**同时通过了两个视角**：产品体验视角（治愈感 + 拟物记忆）+ 业务资产视角（品牌色直接复用 + 跨产品复用 + 国际化潜力）。

## 五、贯穿全产品的 4 条设计原则

1. **场景感知优先**：宁可沉默，也不要在错的时机说对的话（NOMI 教训）
2. **零压力设计**：所有功能可忽略，被拒绝时 AI 不表现失望（动森哲学）
3. **入口轻量，体验可深**：邀请永远是一句话，但接受后可以是 5 分钟深度体验
4. **形象先于功能**：第一秒治愈感 > 功能多强大；**且形象必须沉淀回品牌资产**（v2.0 后新增的边界条件）

## 六、Demo 启动方式

```bash
# V4.1 当前版（推荐）：场景模拟面板 + 10 种形态 + 自动切换
open demo/robotaxi-didi-demo-v4.html

# v3 抽象光滴版（形象重选过渡期）
open demo/robotaxi-didi-demo-v3.html

# v2 猫咪版（决策演进可视化）
open demo/robotaxi-didi-demo.html
```

**推荐演示分辨率**：1920×720（车机主流分辨率）。在 Chrome DevTools → Toggle Device Toolbar → 自定义尺寸 1920×720 即可模拟车机环境。

**V4.1 Demo 已实现的能力**：
- **10 种小桔形态**：6 情绪（idle / happy / curious / listening / sleepy / alert）+ 4 场景化（rainy 雨天 / night 夜晚 / explore 探索 / guardian 守护），每种带独立装饰层（漂浮 emoji / 头顶帽 / 雨滴 / 守护盾等）
- **场景模拟面板**：6 大 Tab × 32 个场景按钮，按钮文案 = 模拟用户语音输入；点击触发 voice / invite / emergency / state 4 种执行流
- **每次对话都有回复**：所有点击都保证至少一条 didi 文字回应，不留对话死角
- **形态自动切换三层机制**：① 待机时每 7s 微切换；② 启动后按时间轴 mock 雨/夜/探索/沉默环境；③ 主动服务弹窗（频率克制）
- **气泡场景化**：颜色 + 图标根据形态自动匹配（雨天蓝 ☂️ / 夜晚暖橙 🌙 / 故宫金 🏯）
- **三选项设计**：「好的 / 你帮我选 / 安静一会儿」，每个邀请都有专属的 accept + auto + silent 三套回复
- **避让交互**：气泡显示时小桔自动下移 + 微缩，气泡消失后回中心，强化"伴侣礼貌让位"感
- **应急离线话术**：≤0.5s 内回应，先事实后安抚（不暗示用户害怕）
- **拒绝零失落**：用户选"安静一会儿"后小桔自然回 idle，不变暗、不卖惨

## 七、技术栈

| 项 | 选型 | 理由 |
|----|------|------|
| 前端框架 | **无**（纯 HTML+CSS+JS）| 零部署成本，浏览器双击即用 |
| 动画引擎 | requestAnimationFrame + 自研弹簧物理 + CSS 变量 lerp | 零依赖，性能可控 |
| 状态管理 | 简单对象 + 函数（10 种形态配置）| 适合此规模，过度抽象反而累赘 |
| 装饰层 | emoji + CSS 关键帧动画 | 跨平台一致 + 可扩展（新场景加 decor 类型即可）|
| 图片策略（V4.1）| 单张 PNG（502KB，已用 OpenCV 中性化处理）| 保留用户喜欢的桔子形象，去除 logo / 文字让 IP 资产中立化 |
| 适配 | 响应式 @media (1500px) | 同时支持 1920×720 和 1920×1080 |
| 部署 | 用户自有 VPS + 宝塔面板 + Let's Encrypt SSL + Nginx Gzip | 国内访问稳定，HTTPS 提升评审信任度 |

详细技术决策过程见 [`03_Agent协作记录/Agent协作全过程记录_v6.0.md` 第四部分](./03_Agent协作记录/Agent协作全过程记录_v6.0.md)（共 6 个独立技术选型决策 + 第九部分实机调优 8 个决策）。

> "**我们希望看到真实的过程，包括遇到的困难和解决方式。**"

我尽力把这件事做扎实了——SVG 失败的 1 小时、PixiJS 失败的 2 小时、AI 推荐 15% 占屏被否决、AI 推荐虚拟人被否决、AI 漏掉应急模块被补充、**我自己推翻了"猫咪"决策**、72 小时主交付完成后**自己作为第一个用户实机操作 1 小时识别 8 个可用性问题并当场修复**——这些"协作中的张力"和"实机调优"都在 `03_Agent协作记录` 和 `04_构建复盘` 里有完整记录。

1. **形象 IP 从「喵喵」改为「小桔」是 72 小时结束前 1 小时做的决策**。我本可以把它"藏起来"——demo 已经做出猫咪了，文档对外宣称"我一开始就选了小桔"也没人能反驳。但我选择把它**完整记录下来**。
