# Robotaxi AI 旅游行程规划助手

这是一个围绕题目“功能一：AI 旅游行程规划助手”实现的可运行 Demo。它模拟乘客在 Robotaxi 座舱内通过语音与 AI 规划未来旅行的过程，重点展示 6 个关键帧：

- 需求发起
- 信息确认
- 方案生成
- 方案调整
- 方案确认
- 保存分享

项目采用 `FastAPI + 原生 HTML/CSS/JavaScript`。后端使用 Python 状态机管理会话、槽位确认和 Mock 行程生成；前端负责把对话与每日行程以座舱大屏形式展示出来。

## 启动方式

当前默认环境为 `conda` 的 `py311`：

```bash
conda activate py311
pip install -r requirements.txt
uvicorn app.main:app --reload
```

启动后访问：

```text
http://127.0.0.1:8000
```

## 测试

```bash
conda activate py311
python -m unittest discover -s tests -p "test_*.py"
```

## 功能说明

### 1. 会话状态机

后端按固定流程推进：

- `initiated`
- `confirming_info`
- `plan_generated`
- `revising_plan`
- `plan_confirmed`
- `shared`

它不是自由对话，而是一个可控的产品原型状态机，保证演示稳定、输出一致、便于评审。

### 2. 槽位确认

首版要求确认以下关键字段：

- 目的地
- 天数
- 出发地
- 出行时间
- 预算

用户如果只说“帮我规划一个云南5天的行程”，系统会继续追问缺失项；补齐后自动生成方案。

### 3. Mock 行程生成

目前以规则引擎生成行程，不接入真实大模型或旅游 API。系统支持：

- 基于预算切换酒店档位
- 基于“慢节奏 / 不要太赶”调整节奏
- 基于“自然风光 / 美食 / 古城”等偏好调整每日亮点
- 基于“加一天大理”等修改意见重排行程

## 演示建议路径

推荐按以下脚本走完整流程：

1. `帮我规划一个云南5天的行程`
2. `我从上海出发，国庆去，预算8000，喜欢自然风光和慢节奏`
3. `预算降到6000，不要太赶，加一天大理`
4. `确认这个方案`
5. `发送到我的手机`

## 技术取舍

- 选择原生前端而非 React/Vue，是为了减少环境依赖和演示启动成本。
- 选择 `FastAPI`，是因为结构清晰，便于快速暴露接口和后续部署。
- 不引入数据库，符合题目 Demo 阶段需求，也避免过度设计。
- 不实现真实语音识别，改用“语音文本输入 + AI 反馈展示”，符合题目要求。
